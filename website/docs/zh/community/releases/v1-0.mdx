# Rsbuild 1.0 发布

## 为什么是 Rsbuild

长期以来，webpack 用户经常困扰于两个问题：**构建性能不佳**和**配置过于复杂**。

Rspack 通过以 Rust 重写 webpack 的方式，很好地解决了构建慢的问题，但为了与现有的 webpack 生态保持兼容，Rspack 仍然保留了 webpack 的配置方式，这意味着 Rspack 依然存在一定的配置复杂度和学习成本。

### 生态的发展

在早期，webpack 生态中出现了一些优秀的工具，比如 Create React App 和 Vue CLI，它们对 React 和 Vue 提供开箱即用的支持，隐藏了复杂的配置细节。因此，React 和 Vue 用户通常更倾向于使用这些工具来创建应用，而不是从零开始配置 webpack。随着生态的发展，Next.js、Nuxt 和 Remix 等全栈 web 框架变得更加流行；Vite 作为一个轻量化的构建工具，也受到了众多开发者的青睐。而 Create React App、Vue CLI 则是逐渐停止了维护。

对于现阶段的 webpack 生态，我们不得不承认一个另人遗憾的事实：webpack 生态缺少一个足够好用的构建工具，它既能像 Create React App 和 Vue CLI 一样简单易用，又能像 Vite 一样具备轻量化、快速启动等特性。

### Rsbuild 的诞生

在一年多以前，我们开始尝试在 Rspack 的基础上，开发一个更现代的构建工具 —— Rsbuild。它以 Rspack 为基础，设计了更易于使用的 API，并内置一套精心设计的构建配置，以充分发挥出 Rspack 的性能优势，并同时解决构建慢和配置复杂的问题。

Rsbuild 主要面向两个使用场景来设计：

- 作为一个轻量的构建工具，帮助开发者快速搭建 web 应用，为 CSR 应用提供开箱即用的支持，也支持通过扩展来支持 SSR、SSG 等模式。
- 作为一个共享的基础设施，为上层工具和框架提供 JavaScript API 和 plugin API，允许更多开发者基于 Rsbuild 来开发属于自己的工具和框架。
